# 競馬予想AI

カミノ競馬クラブ制作。ユーザー自身の予想ロジックを整理・可視化する、Macローカル向けStreamlitアプリです。利益を保証するものではありません。

## 起動

```bash
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
streamlit run app.py
```

OpenAIを使う場合はサイドバーにAPIキーを入力するか、環境変数を設定します。

```bash
export OPENAI_API_KEY="..."
```

キーがない場合も、ルールベースの仮評価で6ステップすべてを試せます。レースJSONは `data/races/`、生成物は `outputs/` に保存されます。

作業中の内容はURL内の下書きIDごとに `data/drafts/` へ自動一時保存されます。ブラウザを閉じたりMacがスリープしても、同じURLを開けば途中から再開できます。確実に残したい予想は、従来通りJSON書き出しまたはSTEP 6の履歴保存も併用してください。

画面は「1 PDF・資料取り込み → 2 Web傾向・出走馬ラップ・AI評価 → 3 スコア・印 → 4 オッズ・直前状態 → 5 買い目 → 6 出力・結果学習」の6段階です。傾向分析とAI評価を同じ画面へまとめ、券種・点数は手入力せず、能力評価を固めてからオッズを確認し、最後に買い方を決める順序にしています。

スマートフォン幅ではサイドバーを自動で閉じ、6手順ナビを横スワイプ式にします。複数列の入力・指標・プランは縦へ積み、評価表と買い目表はカード枠内だけ横スクロールします。入力欄はiPhoneの自動拡大を防ぐ文字サイズ、主要ボタンはタップしやすい高さへ切り替わります。PCとモバイルで同じJSONを利用できます。

画面は競馬情報サイトの見慣れた情報設計を参考に、青いグローバルナビ、白い情報カード、赤い重要表示、淡いグレーの見出し帯で統一しています。外部サイトのロゴ・画像・HTMLは使用していません。

STEP 1ではPDF・PNG・JPEG・WebPも読み込めます。文字を含むPDFはローカル抽出にも対応し、画像やスキャンPDFの解析にはOpenAI APIキーを使います。「高精度解析」ではPDFを高解像度画像へ展開し、長い表を分割・補正してから、文字起こしと構造化を2段階で実施します。STEP 6で「履歴に保存」を選ぶと、予想スナップショットが `data/predictions/` に蓄積されます。

同じ画面レイアウトを毎回使う場合は「保存した固定範囲を使う」を有効にし、レース情報と出馬表の座標範囲を保存できます。APIキーはサイドバーの「このMacに保存」から、Git除外・権限600の `.streamlit/secrets.toml` へ保存でき、次回起動時に自動取得されます。

Macでは、netkeibaのデスクトップ出馬表キャプチャを「MacローカルOCR（無料・推奨）」で解析できます。macOS標準Visionを利用するためOpenAI APIの残高は不要です。OpenAI APIで429/insufficient_quotaが返った場合も、このローカルOCRへ自動で切り替わります。

標準入力は、1ページ縦型の `netkeiba競馬新聞PDF` です。PDFの文字座標からレース概要、馬番、馬名、父、母、母父、性齢、斤量、騎手、厩舎、オッズ、人気、直近5走、通過順位、PDF記載脚質を直接抽出します。脚質表記がない場合は直近走の序盤位置から推定します。

STEP 2の初期値は「キー不要（ローカル）」です。Ollamaとモデルがインストール済みならMac内のローカルAIを自動利用し、未導入なら人気・オッズ・過去走によるローカル仮評価で続行します。OpenAI APIはユーザーが明示的に選択したときだけ使用します。

STEP 2の冒頭では、PDFから取得した全出走馬の直近走を横断集計します。前半3F、上がり3F、S・M・Hのペース記号、通過順位、芝ダート、距離、着順、現在の脚質構成から今回の想定ペースと有利脚質を推定し、各馬へ0〜100の適合指数を付けます。近似条件の好走率、同ペース実績、平均上がりを根拠として表示し、結果をAI・ローカル仮評価の「展開利」と「距離・コース適性」へ反映します。材料が少ない馬は断定せず「条件付き」と表示します。

同じ画面の「Web傾向を取得」では、JRA公式結果から同名レースの過去10年と、同競馬場・芝ダート・距離の過去3年を低頻度で収集します。初回収集はバックグラウンドで進むため、待っている間も他の画面を操作できます。結果は `data/history/aggregates/` に条件別JSONとして保存し、7日以内は再利用します。同名レースは開催月を中心に検索し、開催条件・時期が大きく変更された場合は同条件傾向を優先します。

Web傾向では平均前半3F、平均上がり3F、S・M・Hのペース分布、勝ち馬人気などを整理します。同条件の傾向を同名レースより強めに重み付けし、PDF内の出走馬過去走と照合して「データ推奨馬」を最大5頭表示します。全馬の詳細根拠は折りたたみ内で確認できます。JRA側のページ構成変更や一時的な低速化により取得できない場合も、PDF内データだけで仮評価を続行できます。

サイドバーの「自分の予想方針」は `data/prediction_profile.json` にローカル保存されます。仮評価から手修正した8項目の差分もレースごとに集計し、次回以降のローカルAIまたはOpenAIへ「過去の修正傾向」として渡します。レースデータそのものを学習用に送信する機能ではありません。

STEP 3では8項目の1〜5点入力を増やさず、過去走・人気・オッズによる小さな同点補正を内部で行い、0〜100のレース内指数を表示します。血統は独立採点せず、父・母・母父の情報を距離・コース適性と馬場適性の判断材料へ統合します。ランキングは枠色付きカード、横棒グラフ、全馬比較表で確認できます。

STEP 5では券種や最大点数をユーザーが指定せず、単勝・複勝・枠連・馬連・ワイド・馬単・3連複・3連単をまとめて比較します。予算・最低購入単位・最低買いオッズだけを条件に、「的中重視」「バランス」「高回収狙い」の3案を自動生成し、的中期待指数と回収期待指数の調和がよい案をAIおすすめとして表示します。点数は予算内で内部決定され、採用する案は後から切り替えられます。

STEP 4ではnetkeibaなどの人気上位オッズ表画像をアップロードできます。まずnetkeiba人気上位表向けの固定レイアウトOCRで、馬番・組み合わせ・オッズの数字セルだけを切り出して読み取ります。読み取れない場合は、Macローカル版ではmacOS標準OCR、公開版ではTesseract無料OCRへ切り替え、OpenAI APIキーがある場合だけ画像解析へフォールバックできます。読み取れた人気上位の買い目は取得値として使い、直前オッズを取得できていない券種は、単勝オッズと馬評価から作る推定オッズで比較します。表示する的中期待度・回収期待指数は買い方同士を比較するための参考値で、実際の確率・払戻・利益を保証するものではありません。

同じ画面の「有力馬ステータスカード（オプション）」を開くと、上位6頭の8項目評価をゲーム風の能力値・S〜Fランク・特徴バッジで確認できます。これは新しい評価項目ではなく、現在のスコアの参考表示です。

STEP 6の画像/PDFサマリーは、長い総評・展開・買い判断を枠内で自動省略し、続きはアプリ内で確認できるようにします。全馬評価表は出走頭数に合わせて行高を調整し、最大18頭とレース内指数を1枚に収めます。

レース終了後はSTEP 6下部の「結果・振り返りを登録」から、確定着順、実際の馬場・展開、当たった見解、外れた見解、次回への学び、実購入額・実払戻額を保存できます。勝ち馬の事前順位、上位3頭一致率、評価項目と実際の上位馬との関連を集計し、次回のAI仮評価プロンプトへ反映します。同じレースは学習回数に重複計上しません。

過去にこのツールから書き出した結果付きレースJSONが複数ある場合は、STEP 6の「過去データを一括取り込み」からまとめて選択できます。事前評価・スコア・確定着順が揃ったレースだけを分析し、重複を除いて蓄積します。学習済み件数、勝ち馬の事前平均順位、上位3頭一致率、結果との関連が相対的に強かった評価軸を画面で確認できます。

## JRA指定時刻取得（ベータ）

STEP 4では、PDFから取得した「日付・競馬場・開催回・開催日・レース番号」を使い、JRA公式の公開ページからオッズとレース結果を指定時刻に取得できます。初期値はオッズが発走5分前、結果が発走20分後です。必要な場合だけ、オッズ本取得の3分前に予備取得を1回追加できます。

画面は15秒ごとにローカルの予約時刻だけを確認し、JRAへの通信は予約が実行時刻になった場合だけ行います。各券種ページの取得間隔も2秒以上空けています。予約中はStreamlitアプリを終了せず、ブラウザで開いたままにしてください。Macのスリープ中やアプリ終了中は実行されません。

取得対象は単勝・複勝・枠連・馬連・ワイド・馬単・三連複・三連単です。現行MVPでは単勝オッズを既存の馬評価と買い目再配分へ自動反映し、その他の券種表は取得スナップショットとしてJSONへ保存します。JRA側のページ構成変更や一時的な接続制限に備え、従来の貼り付け入力も残しています。

結果取得に成功すると確定着順を振り返りデータへ記録し、保存済みの事前評価との比較を次回以降の仮評価へ反映します。これはモデル自体を再学習する処理ではなく、このMacに保存した修正・結果傾向を次の評価条件として参照する仕組みです。

## 入力のコツ

出馬表は1頭1行で、例のような並びにすると抽出精度が上がります。

```text
1 1 サンプルホース 牡4 57.0 横山武 美浦・サンプル 3 6.8 先行
2 3 テストランナー 牝5 55.0 ルメール 栗東・テスト 1 3.2 差し
```

抽出結果はSTEP 1の表で自由に修正できます。

## ブラウザで共有する

このリポジトリは、Streamlit Community CloudまたはDocker対応サービスへ公開できます。最も手軽なのはGitHubへこのフォルダを登録し、Streamlit Community Cloudで `app.py` を指定する方法です。Renderを使う場合は同梱の `render.yaml` と `Dockerfile` からウェブサービスを作成できます。

Wi-Fi外のユーザーに共有する場合は、Macの `127.0.0.1` や `192.168.x.x` ではなく、Renderなどが発行する `https://...` の公開URLを使います。具体的な手順は [DEPLOY.md](DEPLOY.md) を参照してください。

公開環境の秘密設定へ `APP_ACCESS_PASSWORD` を登録すると、クラブ内共有用のパスワード画面が有効になります。画像解析にもOpenAIを使う場合だけ、管理者が `OPENAI_API_KEY` を秘密設定へ登録してください。利用者が画面に入力したAPIキーを共有サーバーへ保存するボタンは表示されません。

標準のnetkeiba競馬新聞PDF解析、手修正、スコア、印、買い目、画像・PDF出力はクラウドでも利用できます。macOS標準機能を使う「MacローカルOCR」は公開サーバーでは表示されません。

重くなりすぎないように、アップロード上限は25MB、保存済み予想のサイドバー一覧は最新50件までにしています。Web傾向取得、JRAオッズ取得、画像/PDF出力など重い処理はボタンを押した時だけ動きます。作業途中の下書きはURLごとに自動一時保存されます。

無料の公開環境ではローカルJSONや下書きが再起動時に消える場合があり、同じサーバーの保存履歴は利用者間で共通になります。少人数の試用では各レースのJSON書き出しを併用してください。本格運用で利用者別に履歴を蓄積する場合は、ログイン機能と外部データベースを追加する構成が必要です。また、指定時刻取得は画面を開いているセッションで動作するため、無人の定刻処理には別の定期実行基盤が必要です。
